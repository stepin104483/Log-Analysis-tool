## 2. Analysis of Filtered Bands

### B46 (LTE LAA) - Filtered at NV_Pref
- **Root Cause**: Intentional operator/OEM policy decision
- **Technical Reason**: LAA operates in 5GHz unlicensed spectrum (5150-5925 MHz) and requires Wi-Fi coexistence management
- **Impact**: None for typical deployments; LAA is supplemental downlink only

### n12/n14/n30 (NR SA) - Filtered at HW_Filter
- **Root Cause**: Hardware variant lacks dedicated NR RF paths for these low/mid-bands
- **Technical Reason**: These frequencies overlap with LTE B12/B14/B30 and the RF front-end is configured for LTE-primary operation
- **Impact**: Low - These bands still work via NSA mode where LTE provides the anchor (except n12/n14/n30 which also lack NSA support)

### n12/n14/n25/n30/n48/n70 (NR NSA) - Not in RFC
- **Root Cause**: EN-DC combinations not defined for these bands
- **Technical Reason**: RFC `ca_4g_5g_combos` section doesn't include these band pairings
- **Impact**: 
  - **n25**: Potential gap - n25 SA is supported but NSA is not. If Sprint/T-Mobile network requires EN-DC with n25, this could limit performance
  - **n48**: CBRS - SA-only is common, NSA not critical
  - **n70**: Supplemental band, limited deployment
